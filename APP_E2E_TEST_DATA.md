## 6. 建议的完整验收顺序

### 阶段 A：空库起步

1. 清空业务数据，但保留 `sys_admin_user`
2. 如果你连 `tag_option` 也删了，先执行 [database/03_restore_tag_options.sql](./database/03_restore_tag_options.sql)
3. 用后台 `admin` 登录，确认后台可进

### 阶段 B：注册和登录

1. 用 U01-U06 依次注册
2. 每个账号都至少测试一次：
   - 验证码登录
   - 密码登录
   - 退出登录
3. U05 只注册并完善资料，不发内容，用来验证空状态

### 阶段 C：后台基础内容

1. 创建 A01-A02 公告
2. 创建 T01-T02 活动
3. 创建 M01-M03 商家
4. 验证 M03 在“意向商家”状态下不能用于创建合作单
5. 把 M03 设为合作中
6. 创建 C01-C03

### 阶段 D：前台内容链路

1. U01 发布 P01
2. U02 发布 P02
3. U03 发布 P03
4. U05 新建 P05 草稿但不发布
5. U06 暂不发合作内容，先登录看看“我的合作”空/待确认状态

### 阶段 E：后台审核

1. P01 审核通过
2. P02 审核通过
3. P03 第一次驳回
4. U03 修改后重发
5. P03 第二次审核通过

### 阶段 F：合作链路

1. U01 进入“我的合作”，接受 C01
2. U03 进入“我的合作”，拒绝 C02
3. U06 接受 C03
4. U01 发布 P06，并绑定 C01
5. U06 发布 P04，并绑定 C03
6. 后台审核 P06、P04 通过
7. U02/U03/U04 对 P06 完成互动，使 C01 达标
8. C01 应进入“待发奖励”
9. `finance` 发放 C01 奖励
10. U01 在激励中心申请提现
11. `finance` 审核通过 U01 提现

### 阶段 G：活动 / 搜索 / 互动 / 个人中心

1. 用 U02、U04 测活动报名、取消报名、我的活动
2. 用 U04 测搜索：
   - 关键词：`图书馆`
   - 关键词：`社团`
   - 组合筛选：`场景=图书馆 / 风格=学院风 / 预算=100-150`
3. 用 U04 测详情页点赞、收藏、评论、商品跳转
4. 用 U01/U02/U04 测关注列表、粉丝列表
5. 用 U05 测个人页空状态、编辑资料、消息空状态
6. 用 U01 测：
   - 我的内容
   - 我的合作
   - 激励中心
   - 提现记录
   - 消息通知

## 9. 最后一个建议

这次验收不要追求“把所有账号都玩得很满”，而是按角色走：

- U01：跑主链路
- U03：跑异常链路
- U05：跑空状态链路
- U06：跑未达标合作链路
- U02/U04：给别人制造互动数据

这样最省时间，也最容易看出真 bug。
