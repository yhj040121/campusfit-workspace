# 青搭毕业设计说明书框架建议

## 1. 参考文档格式梳理

根据 `201603090225徐文特毕业设计.pdf`，该类毕业设计说明书的典型结构为：

1. 封面
2. 诚信承诺书
3. 目录
4. 中文摘要、关键词
5. 英文摘要、关键词
6. 正文
7. 参考文献
8. 致谢
9. 附录

正文通常采用 5 章结构：

### 第 1 章 绪论

- 研究意义
- 研究现状
- 可行性分析
- 主要研究内容

### 第 2 章 项目需求分析

- 调研方式或需求来源
- 用户画像分析
- 功能需求分析

### 第 3 章 系统整体结构设计

- 系统整体结构
- 主要业务流程
- 功能模块设计
- 数据库设计

### 第 4 章 系统开发实现

- 关键技术
- 开发环境
- 前后端实现
- 具体功能模块实现

### 第 5 章 总结与展望

- 总结
- 展望

## 2. 你的项目适合采用的写法

你的项目“青搭”与参考文档同样属于“产品方案 + 系统设计 + 开发实现”类型，因此非常适合沿用这个五章结构，但需要把原文中的“冷门宠物 App”替换为“校园穿搭内容与导购平台”语境。

你的现有材料已经具备以下可写基础：

- 产品定位与背景
- 用户角色与业务闭环
- 用户端与后台功能结构
- Web、移动端、后台、数据库的实际工程文件
- 活动、激励、商家合作、审核治理等业务规则

因此，你的说明书不需要从零开始写，而是要把现有 PRD/项目文档改写为更偏“毕业设计论文表达”的结构。

## 3. 青搭建议章节框架

下面是一版可直接用于写作的目录框架。

### 封面

建议题目：

`基于 Vue 3 与 Spring Boot 的青搭校园穿搭内容平台设计与实现`

也可以写成：

`校园穿搭内容与导购平台青搭的设计与实现`

### 摘要

摘要建议包含 4 个要素：

- 项目背景与问题
- 系统目标与核心功能
- 采用的技术方案
- 最终实现效果

关键词建议：

- 校园穿搭
- 内容平台
- Vue 3
- Spring Boot
- MySQL

### 第 1 章 绪论

#### 1.1 研究背景与意义

可围绕以下内容展开：

- 大学生存在真实穿搭参考需求
- 现有平台内容分散，缺少校园垂直场景聚合
- 电商平台强交易但弱内容决策支持
- 校园穿搭内容平台具备产品价值与实现价值

#### 1.2 相关研究与行业现状

建议拆成：

- 1.2.1 校园内容平台或垂直社区的发展现状
- 1.2.2 穿搭内容平台与导购模式的发展现状
- 1.2.3 Web 与移动端前后端分离开发模式的发展现状

#### 1.3 可行性分析

建议拆成：

- 1.3.1 技术可行性分析
- 1.3.2 业务可行性分析
- 1.3.3 经济与实施可行性分析

这里可以结合你的实际项目说明：

- 前端采用 Vue 3 + Vite
- 后端采用 Spring Boot 3
- 数据库采用 MySQL 8
- 移动端采用 uni-app
- 项目已经具备原型、前端、后台、数据库和测试环境

#### 1.4 本文主要研究内容

建议写成：

- 对校园穿搭平台的需求进行分析
- 设计系统的业务流程、功能结构和数据库结构
- 实现用户端、管理后台与后端接口
- 对系统落地效果进行总结，并提出后续优化方向

### 第 2 章 系统需求分析

#### 2.1 需求来源与目标用户分析

可以写：

- 需求来源于大学生真实校园穿搭场景
- 核心用户包括普通大学生、预算敏感型用户、校园创作者
- 平台角色包括运营、审核、商家、管理员

#### 2.2 用户痛点与核心场景分析

建议覆盖：

- 内容分散，缺少校园场景聚合
- 预算敏感，希望获得低决策成本参考
- 创作者需要曝光、互动和激励
- 商家需要校园精准曝光场景

#### 2.3 功能需求分析

建议拆成：

- 2.3.1 用户端功能需求
- 2.3.2 管理后台功能需求
- 2.3.3 平台运营与激励功能需求

用户端可重点写：

- 登录注册与资料完善
- 首页内容流
- 内容详情、点赞、评论、收藏、关注
- 搜索与标签筛选
- 内容发布与草稿箱
- 活动参与与消息通知
- 个人中心与激励中心

后台可重点写：

- 内容审核
- 活动管理
- 公告管理
- 商家管理
- 结算与提现审核
- 数据看板

#### 2.4 非功能需求分析

建议增加这一节，增强论文感：

- 易用性
- 可维护性
- 可扩展性
- 安全性
- 性能与稳定性

### 第 3 章 系统整体结构设计

#### 3.1 系统总体架构设计

建议画出系统总体架构图，说明：

- 移动端 `mobile-uniapp`
- Web 用户端 `web`
- 管理后台 `admin-web`
- 后端服务 `backend-java`
- 数据库 `database`

#### 3.2 系统核心业务流程设计

- 3.2.3 内容发布与审核流程
- 3.2.4 活动参与与激励流程

#### 3.3 功能模块结构设计

- 3.3.2 管理后台功能结构设计

#### 3.4 数据库设计

建议拆成：

- 3.4.1 数据库概念设计
- 3.4.2 主要数据表设计
- 3.4.3 核心表关系说明

可重点选择以下表进行说明：

### 第 4 章 系统开发与实现

#### 4.1 系统开发关键技术

建议拆成：

- 4.1.1 Vue 3 与 Vite 前端框架
- 4.1.2 uni-app 移动端方案
- 4.1.3 Spring Boot 3 后端框架

- `web/`

- 用户端页面实现

建议作为重点章节，可写：

- 本地开发环境搭建
- 前后端联调方式
- 数据初始化方式
- 关键流程测试结果
- 已知问题与处理方式

### 第 5 章 总结与展望

#### 5.1 总结

总结建议回答三个问题：

- 这个系统解决了什么问题
- 完成了哪些核心模块
- 本次设计与实现的主要成果是什么

#### 5.2 展望

可写后续方向：

- 优化推荐与内容分发机制
- 完善创作者成长体系
- 增强商家合作与活动玩法
- 增加更完整的数据分析能力

## 5. 建议你优先采用的真实写作顺序

为了降低写作难度，建议按下面顺序整理：

1. 先写第 2 章需求分析，因为你的现有材料最完整
2. 再写第 3 章总体设计，把流程图、功能结构图、数据库表补齐
3. 接着写第 4 章系统实现，按前端、后台、数据库拆开
4. 最后补第 1 章绪论和第 5 章总结展望
5. 摘要放在最后统一提炼

## 6. 写作上的改写提醒

你现在很多材料更偏 PRD 和项目文档表达，改成毕业设计说明书时要注意：

- 少用“我要做什么”，多用“本文设计了什么”
- 少用口语化产品表达，多用客观描述
- 每章尽量配图，例如架构图、流程图、ER 图、页面图
- 第 4 章不要只写页面名称，要写实现逻辑和技术方案
- 第 5 章不要只写“完成了项目”，要总结设计价值、实现成果和不足
