# 09 MVP 范围与版本规划

## 1. 文档目标

明确什么必须先做、什么可以后做，控制项目范围，确保青搭能按阶段推进。

## 2. MVP 设计原则

- 先验证内容消费和导购路径能否成立。
- 先保证供给、消费、互动、审核四条链路跑通。
- 后做复杂商业化和复杂运营系统。

## 3. MVP 范围

### 3.1 必做能力

- 登录注册与基础资料
- 首页推荐流
- 内容详情页
- 点赞、评论、收藏、关注
- 搜索与标签筛选
- 发布内容
- 商品链接挂载与跳转
- 草稿箱
- 消息通知
- 后台内容审核
- 后台商品链接审核
- 基础数据看板

### 3.2 可延后能力

- 创作者等级体系
- 商家合作全流程管理
- 结算系统
- 复杂推荐算法
- 高级运营活动配置

## 4. 版本建议

### 4.1 V1：闭环验证版

目标：

- 跑通“内容浏览 -> 互动 -> 导购跳转 -> 回访”的基础链路。

范围：

- 用户端基础消费和创作能力
- 审核后台
- 消息中心

### 4.2 V2：运营增强版

目标：

- 提升内容分发效率和活动运营能力。

范围：

- 专题活动
- 更完整的搜索筛选
- 创作者管理
- 更细的数据看板

### 4.3 V3：商业化拓展版

目标：

- 验证合作和变现模式。

范围：

- 商家管理
- 合作资源位
- 分佣或结算支持

## 5. 优先级排序建议

| 优先级 | 模块 |
| --- | --- |
| P0 | 内容消费、详情、发布、标签、导购、审核 |
| P1 | 消息通知、专题活动、举报治理、创作者管理 |
| P2 | 商家合作、结算、深度数据分析、复杂推荐 |

## 6. 里程碑建议

- 里程碑一：前后台核心流程图与 PRD 定稿
- 里程碑二：用户端主链路原型与后台审核方案完成
- 里程碑三：核心功能开发联调完成
- 里程碑四：上线前验收、数据埋点检查、灰度方案准备

## 7. 风险控制

- 若资源有限，优先保证详情页和发布页质量。
- 若内容供给不足，优先引入运营精选或平台示范内容。
- 若后台能力不足，先收缩对外开放范围，避免治理失控。
