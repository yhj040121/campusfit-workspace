# 02 目标用户与角色定义

## 1. 文档目标

定义青搭的关键用户和平台参与角色，明确不同角色的目标、行为和产品支持重点。

## 2. 核心用户群体

### 2.1 普通校园用户

特征：

- 18 到 24 岁在校大学生
- 有穿搭参考需求，也有预算敏感特征
- 更偏好真实、可复用、可模仿的内容

核心诉求：

- 快速找到适合校园场景的穿搭灵感
- 根据预算、风格和场景筛选内容
- 从内容直接获取商品信息

### 2.2 预算理性型用户

特征：

- 注重性价比
- 关注单品复用率、预算范围和日常可穿性

核心诉求：

- 降低冲动消费
- 看到真实价格带和高复用搭配建议
- 更快做出购买决策

### 2.3 校园内容创作者

特征：

- 乐于分享穿搭内容
- 关注曝光、互动、粉丝增长和合作机会

核心诉求：

- 更方便发布内容和挂载商品
- 获得稳定曝光与互动反馈
- 被平台识别为可合作对象

## 3. 平台角色

### 3.1 运营人员

负责内容活动、专题配置、消息触达、校园主题策划与基础数据复盘。

### 3.2 审核人员

负责内容审核、商品链接审核、举报处理和违规治理。

### 3.3 商务或商家合作方

负责商家入驻、商品链接维护、活动合作和投放资源协调。

### 3.4 管理员

负责权限管理、系统配置、风险升级处理和数据总览。

## 4. 角色与核心任务映射

| 角色 | 核心任务 | 关键页面或能力 |
| --- | --- | --- |
| 普通校园用户 | 浏览内容、搜索场景、互动、收藏、导购跳转 | 首页、详情、搜索、个人中心 |
| 预算理性型用户 | 筛选预算友好内容、查看高复用单品 | 搜索、标签、详情 |
| 校园创作者 | 发布穿搭、打标签、挂商品、查看互动反馈 | 发布页、草稿箱、我的发布 |
| 运营人员 | 配置活动、推送消息、观察数据 | 后台控制台、专题、消息配置 |
| 审核人员 | 审核内容、处理举报、下架违规内容 | 审核中心、治理中心 |
| 商务或商家合作方 | 管理商品链接、活动合作 | 商家管理、导购管理 |
| 管理员 | 分配权限、查看全站数据、处理高风险问题 | 系统设置、数据看板 |

## 5. 优先服务顺序

建议的角色优先级：

1. 普通校园用户
2. 校园内容创作者
3. 运营与审核人员
4. 商家合作方

原因：

- 没有稳定内容消费和互动，商业化基础无法成立。
- 创作者供给是内容平台冷启动的关键。
- 后台治理和运营能力是平台可持续运行的基础保障。

## 6. 设计原则

- 对用户：减少选择成本，增强场景感和可信度。
- 对创作者：降低发布成本，提高反馈速度。
- 对运营：确保可配置、可观察、可回溯。
- 对审核：规则明确、流程可追踪、处罚可申诉。
