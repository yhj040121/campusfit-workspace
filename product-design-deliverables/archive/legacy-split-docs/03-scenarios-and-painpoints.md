# 03 使用场景、痛点与机会点

## 1. 文档目标

把“用户为什么来、在什么时刻来、要解决什么问题”说清楚，作为流程与功能设计的起点。

## 2. 关键使用场景

### 2.1 日常校园出行场景

典型触发：

- 早八上课
- 图书馆自习
- 社团活动
- 校园散步或拍照

用户目标：

- 快速找到不出错、适合当下场景的穿搭参考

### 2.2 预算有限但想提升搭配效率的场景

典型触发：

- 手头预算有限
- 想避免重复购买低利用率单品
- 想找价格友好且搭配效果稳定的单品

用户目标：

- 用更低成本完成更合理的搭配决策

### 2.3 内容创作者日常分享场景

典型触发：

- 想记录今日穿搭
- 想参与平台活动或校园专题
- 想提升个人曝光与互动

用户目标：

- 低门槛发布，高效率获得反馈和增长

### 2.4 运营治理场景

典型触发：

- 内容增多后需要审核与巡检
- 商品链接需要验证有效性和合规性
- 举报与违规问题需要闭环处理

用户目标：

- 控制风险，维持社区质量，提升运营效率

## 3. 主要痛点

### 3.1 普通用户痛点

- 校园穿搭内容分散，缺乏垂直聚合。
- 主流内容平台偏“种草”，但不够聚焦校园具体场景。
- 主流电商平台偏交易，不擅长通过内容解决穿搭决策问题。
- 很多内容好看但不实穿，参考价值不稳定。

### 3.2 预算理性型用户痛点

- 缺少以预算、复用率和实穿性为导向的推荐方式。
- 容易被高颜值内容带偏，缺少理性筛选依据。

### 3.3 创作者痛点

- 发布内容后难以获得精准分发。
- 合作机会、活动入口和成长反馈不清晰。
- 内容与商品导购能力割裂。

### 3.4 平台痛点

- 内容、互动、导购和运营之间容易脱节。
- 没有后台治理能力时，平台增长越快风险越大。
- 若没有结构化标签和场景体系，推荐和搜索效率会下降。

## 4. 机会点总结

| 机会点 | 对应策略 |
| --- | --- |
| 校园场景化内容有明显空白 | 做场景标签体系和专题内容运营 |
| 用户决策成本高 | 强化标签、搜索、预算筛选和详情页信息密度 |
| 创作者增长路径不清晰 | 建立创作者成长、活动、消息反馈机制 |
| 导购价值难沉淀 | 建立内容到商品的结构化导购路径 |
| 平台治理不足 | 补齐审核、举报、下架、申诉和数据回溯流程 |

## 5. 优先级建议

- P0：场景化内容浏览、详情互动、发布、搜索、标签、导购跳转
- P1：消息通知、草稿箱、举报治理、专题活动
- P2：创作者分层、商家合作管理、结算与更完整的数据分析

## 6. 结论

CampusFit 的核心机会不在“再做一个时尚社区”，而在“用校园真实场景和导购效率，重构大学生穿搭决策流程”。
