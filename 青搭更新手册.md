# 青搭更新手册

适用范围：

- 管理后台前端：`E:\test\admin-web`
- Java 后端：`E:\test\backend-java`
- 管理后台域名：`https://admin.yuhaojie.cn`
- API 健康检查：`https://admin.yuhaojie.cn/api/health`
- 后端服务名：`campusfit-backend`
- 服务器公网 IP：`47.76.140.208`

---

## 1. 现在这套部署关系

- 前端静态文件部署在服务器：`/var/www/qingda-admin/dist`
- Nginx 对外提供：`https://admin.yuhaojie.cn`
- Nginx 反向代理：`/api/* -> http://127.0.0.1:8080`
- Java 后端通过 `systemd` 管理，服务名：`campusfit-backend`

---

## 2. 平时更新的标准顺序

每次更新，推荐按这个顺序走：

1. 本地修改代码
2. 本地构建前端
3. 本地打包后端
4. 上传前端构建产物
5. 上传后端 jar
6. 重启后端服务
7. 如果改了 Nginx，再重载 Nginx
8. 打开后台和健康检查验证

---

## 3. 前端更新流程

### 3.1 本地构建

Windows PowerShell:

```powershell
cd E:\test\admin-web
npm install
npm run build
```

构建完成后，产物在：

- `E:\test\admin-web\dist`

### 3.2 服务器备份旧前端

登录服务器：

```bash
ssh root@47.76.140.208
```

备份旧版本：

```bash
sudo mkdir -p /var/www/qingda-admin
sudo mv /var/www/qingda-admin/dist /var/www/qingda-admin/dist.bak-$(date +%Y%m%d-%H%M%S) 2>/dev/null || true
```

### 3.3 上传新前端

回到本地 Windows PowerShell：

```powershell
scp -r E:\test\admin-web\dist root@47.76.140.208:/var/www/qingda-admin/
```

### 3.4 修复权限

回到服务器：

```bash
sudo chown -R www-data:www-data /var/www/qingda-admin/dist
```

### 3.5 验证前端

```bash
curl -I https://admin.yuhaojie.cn/
curl -s https://admin.yuhaojie.cn/ | head -20
```

如果你改的是纯前端页面，通常到这里就够了，不需要重启后端。

---

## 4. 后端更新流程

### 4.1 本地打包

Windows PowerShell:

```powershell
cd E:\test\backend-java
mvn clean package -DskipTests
```

默认产物：

- `E:\test\backend-java\target\campusfit-backend-0.0.1-SNAPSHOT.jar`

### 4.2 先确认服务器真实 jar 路径

服务器执行：

```bash
sudo systemctl cat campusfit-backend | grep ExecStart
```

记住输出里的 jar 路径，例如可能类似：

```bash
/srv/campusfit/backend/backend-java/target/campusfit-backend-0.0.1-SNAPSHOT.jar
```

### 4.3 上传新 jar

本地 Windows PowerShell：

```powershell
scp E:\test\backend-java\target\campusfit-backend-0.0.1-SNAPSHOT.jar root@47.76.140.208:/tmp/
```

### 4.4 替换服务器上的 jar

服务器执行。把下面目标路径改成你在 `ExecStart` 里看到的真实路径：

```bash
sudo cp /tmp/campusfit-backend-0.0.1-SNAPSHOT.jar /srv/campusfit/backend/backend-java/target/campusfit-backend-0.0.1-SNAPSHOT.jar
```

### 4.5 重启后端

```bash
sudo systemctl restart campusfit-backend
sudo systemctl status campusfit-backend --no-pager
journalctl -u campusfit-backend -n 100 --no-pager
```

### 4.6 验证后端

```bash
curl http://127.0.0.1:8080/api/health
curl -i https://admin.yuhaojie.cn/api/health
```

正常时应该返回 JSON，而不是 HTML 页面。

## 5. 如果改了数据库

先备份数据库：

```bash
mysqldump -u root -p campusfit > /root/campusfit-$(date +%Y%m%d-%H%M%S).sql
```

执行 SQL：

```bash
mysql -u root -p campusfit < /path/to/your_patch.sql
```

如果这次后端依赖新表结构，导入完成后再重启后端：

```bash
sudo systemctl restart campusfit-backend
```

---

## 6. 如果改了 Nginx

当前建议使用的管理后台配置文件可以放在：

- `/etc/nginx/sites-available/00-admin.yuhaojie.cn.conf`

修改完 Nginx 后执行：

```bash
sudo nginx -t
sudo systemctl reload nginx
```

常用检查：

```bash
sudo grep -Rni "server_name .*admin.yuhaojie.cn" /etc/nginx/sites-enabled /etc/nginx/sites-available /etc/nginx/conf.d
```

## 8. 回滚方法

### 8.1 前端回滚

先看备份：

```bash
ls -la /var/www/qingda-admin/
```

回滚某个备份：

```bash
sudo rm -rf /var/www/qingda-admin/dist
sudo mv /var/www/qingda-admin/dist.bak-20260326-000000 /var/www/qingda-admin/dist
sudo chown -R www-data:www-data /var/www/qingda-admin/dist
```

### 8.2 后端回滚

如果你保留了旧 jar，就把旧 jar 再复制回原路径，然后重启：

```bash
sudo cp /path/to/old.jar /你的真实jar路径/
sudo systemctl restart campusfit-backend
```

## 9. 故障排查

### 9.1 后台打开了，但接口不通

先看是不是 HTML 而不是 JSON：

```bash
curl -i https://admin.yuhaojie.cn/api/health
```

如果返回 HTML，说明 Nginx 的 `/api/` 代理没生效。

### 9.2 后端服务起不来

```bash
sudo systemctl status campusfit-backend --no-pager
journalctl -u campusfit-backend -n 200 --no-pager
```

### 9.3 Nginx 配置冲突

```bash
sudo grep -Rni "server_name .*admin.yuhaojie.cn" /etc/nginx/sites-enabled /etc/nginx/sites-available /etc/nginx/conf.d
```

### 9.4 前端还是旧页面

先检查服务器上是不是新包：

```bash
grep -n "青搭管理后台" /var/www/qingda-admin/dist/index.html
```

如果搜不到，说明上传的还是旧包。

## 10. 推荐习惯

- 前端每次上传前都先 `npm run build`
- 后端每次上传前都先 `mvn clean package -DskipTests`
- 改数据库前先备份
- 改 Nginx 后必须 `nginx -t`
- 每次上线后至少验证这三个地址：
  - `https://admin.yuhaojie.cn/`
  - `https://admin.yuhaojie.cn/api/health`
  - `http://127.0.0.1:8080/api/health`
